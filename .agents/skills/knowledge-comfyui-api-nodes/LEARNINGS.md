## 2026-06-23 — Nós Vídeo→Vídeo + os 2 padrões de saída (do `/object_info` ao vivo)
- **Contexto**: criados os bundles `workflows-api/image-to-video-api/` (8 modelos I2V) e `video-to-video-api/` (restyle/motion-transfer/extend). Schemas tirados do `/object_info` ao vivo do `:8188`.
- **Aprendizado**: (1) **2 padrões de saída** governam a fiação E o áudio: **fal `*_fal`** → `STRING/video_url` → `LoadVideoURL→CreateVideo→SaveVideo` (perde áudio); **partner** (Comfy credits) → `VIDEO` nativo → `SaveVideo` direto (áudio ok). (3) V2V confirmados instalados: `RunwayAleph2VideoToVideoNode` (restyle, vídeo 2–30s, partner — saída VIDEO), `GrokVideoEditNode` (≤8.7s/50MB), `KlingOmniVideoToVideoEdit_fal` (edit+elementos), `KlingV3Pro/StandardMotionControl_fal`, `ViduExtendVideoNode`/`GrokVideoExtendNode` (extend de **arquivo**) vs `KlingVideoExtendNode` (só `video_id`, não arquivo). (5) `ByteDanceImageToVideoNode` (partner) traz Seedance 1.5-pro com 1080p+`generate_audio`. (6) `output_name`/`output` por nó **devem** ser lidos do `/object_info` p/ saber A vs B (não dá p/ adivinhar pelo nome).

## 2026-06-23 — `FluxProKontextMulti_fal`: schema exato (widgets) + seed gate confirmado
- **Contexto**: montei `workflows-api/replace-object` e `replace-environment`; precisei do schema EXATO de `FluxProKontextMulti_fal` (não havia exemplo no repo) — puxado do `/object_info` do ComfyUI vivo.
- **Aprendizado**: (1) `FluxProKontextMulti_fal` — required: `prompt`(STRING), `image_1`, `image_2`(IMAGE); optional: `image_3`, `image_4`(IMAGE), `aspect_ratio`(None|21:9..9:21, default None=mantém input), `max_quality`(bool=false), `guidance_scale`(3.5), `num_images`(1), `safety_tolerance`('1'..'6'='2'), `output_format`(jpeg|png), `sync_mode`(false), `seed`(0). **Ordem dos `widgets_values`**: prompt, aspect_ratio, max_quality, guidance_scale, num_images, safety_tolerance, output_format, sync_mode, seed, **`control_after_generate`('fixed')**. (2) **Seed gate confirmado**: `seed=0` + control `fixed`; control `randomize`/`increment` faria seed>0 e TRAVARIA (consistente com FluxPro1Fill que usa seed + 'fixed' nos widgets). (3) `NanoBananaPro_fal`: `images`(IMAGE batch), `prompt`, `num_images`(1), `aspect_ratio`(default '1:1', aceita 'auto'), `output_format`(png), `resolution`(1K/2K/4K), `sync_mode`; **sem seed**. (4) **Não existe** nó de API que aceite máscara + imagem-de-referência juntos → para "inserir um objeto fornecido como imagem numa região", use multi-ref (Nano/Kontext) full-frame + composite mascarado LOCAL (`ImageCompositeMasked` com `mask_inverted`).
- **Fonte**: usuário (tarefa) + `/object_info` do servidor local.
- **Ação**: corpo já cobre o seed gate do Kontext; manter aqui o schema de widgets p/ reconstruir o nó sem a UI. Revalidar se `ComfyUI-fal-API` atualizar.
